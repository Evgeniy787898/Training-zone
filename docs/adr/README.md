# Architecture Decision Records (ADR)

Этот каталог содержит архитектурные решения проекта TZONA в формате ADR.

## Что такое ADR?

ADR (Architecture Decision Record) — это краткий документ, фиксирующий важное архитектурное решение с контекстом и обоснованием.

## Формат

Каждый ADR следует шаблону:

```markdown
# ADR-NNN: Название решения

**Статус:** Принято | Отклонено | Заменено ADR-XXX
**Дата:** YYYY-MM-DD
**Авторы:** @username

## Контекст
Какая проблема или потребность привела к этому решению?

## Решение
Что именно решили сделать?

## Альтернативы
Какие варианты рассматривались и почему были отклонены?

## Последствия
Какие плюсы и минусы несёт это решение?

## Связанные документы
- Ссылки на связанные ADR, issues, документацию
```

## Как добавить новый ADR

1. Скопируйте шаблон из `000-template.md`
2. Назовите файл `NNN-короткое-название.md`
3. Заполните все секции
4. Добавьте запись в индекс выше
5. Создайте PR для ревью
