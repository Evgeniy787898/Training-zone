# Database Index Audit

This report is generated by `npm run audit:indexes --prefix backend`, which connects to the Postgres database defined by `DATABASE_URL` and inspects `pg_stat_user_indexes`/`pg_stat_user_tables` to highlight unused, rarely used, and large indexes, plus any tables missing indexes.

## How to regenerate

1. Ensure `DATABASE_URL` points to the target Postgres instance (e.g., production replica or staging database).
2. Run `npm run audit:indexes --prefix backend` after loading `backend/.env` (or export `DATABASE_URL` manually).
3. Commit the refreshed `docs/database-index-audit.md` so the latest snapshot is tracked alongside the codebase.
